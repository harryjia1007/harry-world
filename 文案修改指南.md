# 自己改網站文案 — 三步驟（給 Harry）

> 結論：**文字幾乎都在 `app.js` 最上面那一段**，用任何文字編輯器改完存檔，
> 貼一行指令部署，就上線了。不用懂程式。

## 第 1 步：打開檔案改字

用「文字編輯 TextEdit」或 VS Code 打開：
`~/Desktop/專案/harry-world/app.js`

檔案**最上面**就是所有內容區（往下捲一點點就看到）：

| 想改什麼 | 找哪個區塊 | 長什麼樣 |
|---|---|---|
| 作品卡片（名稱/描述/狀態） | `const WORKS = [` | `name: 'NotchGlass',` |
| 經歷時間軸 | `const JOURNEY = [` | `title: 'Aiii.Ai — BD 實習生',` |
| 生活照卡片的小標 | `const STRIP = [` | `cap: '01 ✦ life'` |
| 「目前正在…」三行 | `const CURRENTLY = [` | `'打磨 NotchGlass 中',` |
| 聯絡方式 | `const CONTACTS = [` | `label: 'EMAIL',` |
| 關於我的標籤 chips | `const ABOUT_CHIPS = [` | |

**改字的規則只有兩條：**
1. 只改**引號裡面**的字：`'這裡的字可以隨便改'` — 引號本身、逗號、括號都不要動。
2. 文字裡如果要用到單引號 `'`，先問我（會讓程式壞掉）。

另外「嗨，我是陳佳朋」那段自我介紹在 `index.html` 裡（搜尋你想改的那句話就找得到），規則相同。

## 第 2 步：本機先看一眼（可跳過）

直接雙擊打開 `index.html` 就能在瀏覽器預覽。

## 第 3 步：部署上線（一行，複製貼到終端機）

```
cd ~/Desktop/專案/harry-world && npx wrangler deploy
```

看到 `Deployed harry-world triggers` 加 `harryjia.com` 就是成功了，
去 harryjia.com 重新整理確認（手機可能要強制重新整理）。

## 改壞了怎麼辦

不要慌，所有版本都在 git 裡。跟我說「網站改壞了幫我還原」即可；
或自己跑：`cd ~/Desktop/專案/harry-world && git checkout app.js`（把 app.js 退回上次提交的狀態）。
